# Product Specs Index

Product specs define what we're building from the user's perspective.
They are the source of acceptance criteria for ExecPlans.

| Spec | Status | Summary |
|---|---|---|
| [audiobook-generation.md](audiobook-generation.md) | Active | Core audiobook generation flow |

## How to use product specs

When writing an ExecPlan, reference the relevant product spec for acceptance
criteria. The spec defines what "done" looks like from a user perspective.
The ExecPlan defines how to get there.
